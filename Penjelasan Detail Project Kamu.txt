Koneksi database
Di setiap file kamu membuat koneksi seperti ini:
<?php
$dbHost = 'localhost';
$dbName = 'latihanform';
$dbUser = 'root';
$dbPass = '';

$pdo = new PDO("mysql:host=$dbHost;dbname=$dbName;charset=utf8mb4", $dbUser, $dbPass);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

Artinya:

PDO = interface untuk MySQL
charset=utf8mb4 = supaya dukung karakter Indonesia
ERRMODE_EXCEPTION = supaya error muncul sebagai exception

3. Registrasi: register.php
Alur / algoritma jadi:
user buka halaman
$_SERVER["REQUEST_METHOD"] == "POST" = cek apakah form disubmit
ambil input:
name
email
password
confirm_password
validasi:
apakah kosong?
apakah email valid?
apakah email sudah terdaftar?
apakah password cukup panjang?
apakah konfirmasi password cocok?
jika semua valid:
password_hash($password, PASSWORD_DEFAULT)
simpan ke database
redirect ke login.php?registered=1
Syntax penting
trim($_POST["name"]) → hapus spasi di awal/akhir
filter_var($email, FILTER_VALIDATE_EMAIL) → validasi email
password_hash(..., PASSWORD_DEFAULT) → enkripsi password
header("Location: ...") → redirect
exit; → hentikan eksekusi setelah redirect

6. Proteksi Halaman dengan Session
Semua halaman penting memakai pola ini:
<?php
session_start();
if (!isset($_SESSION['user'])) {
    header("Location: login.php");
    exit;
}

Itu artinya:

sebelum halaman bisa dibuka, user harus login
kalau belum login, langsung diarahkan ke login.php
Ini penting supaya tidak bisa masuk ke dashboard.php, profile.php, atau attendance.php tanpa otentikasi.

Kenapa perlu query ulang user dari DB?
Karena jika user baru saja upload foto, data di session perlu diperbarui atau mengambil dari DB agar tampil terbaru. Di profile.php kamu secara langsung memanggil DB lagi supaya profile_photo terbaru muncul.

Syntax penting

enctype="multipart/form-data" → wajib pada form file upload

move_uploaded_file($file['tmp_name'], __DIR__ . '/' . $targetPath) → simpan file
isset($user['profile_photo']) && $user['profile_photo'] ? ... : ...
→ jika tidak ada foto, pakai placeholder


8. Absensi: attendance.php
Tujuan
Membuat sistem presensi harian:

check-in
check-out
riwayat presensi 7 hari terakhir
Algoritma
cek login
ambil today = date('Y-m-d')
jika tombol check_in diklik:
cari record attendance untuk user + tanggal hari ini
kalau sudah ada → tidak boleh check-in lagi
kalau belum ada → INSERT record baru dengan check_in_time
jika tombol check_out diklik:
cari record hari ini
jika belum check-in → beri pesan belum bisa check-out
jika sudah check-out sebelumnya → beri pesan sudah pulang
jika sudah check-in tetapi belum check-out → UPDATE check_out_time
tampilkan data hari ini dan riwayat
Syntax penting
date('Y-m-d') → tanggal hari ini
date('H:i:s') → waktu saat ini
SELECT * FROM attendance WHERE user_id = :user_id AND attendance_date = :date
INSERT INTO attendance (...) VALUES (...)
UPDATE attendance SET check_out_time = :time WHERE id = :id
fetch(PDO::FETCH_ASSOC) → ambil satu baris
fetchAll(PDO::FETCH_ASSOC) → ambil banyak baris

11. Kunci Pemahaman yang Harus Kamu Catat
a. Session = kunci login
session_start() harus di setiap halaman login-protected
$_SESSION['user'] menyimpan data user
b. Prepared statement = aman dari SQL injection
prepare(...) + execute([...])
contoh:
<?php
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute([':email' => $email]);

c. Password harus di-hash
password_hash() menyimpan password aman
password_verify() membandingkan input dengan hash
d. File upload butuh enctype="multipart/form-data"
jika tidak, file tidak akan dikirim
data file masuk ke $_FILES
e. Jangan lupa validasi
input kosong
format email
ukuran file
tipe file
